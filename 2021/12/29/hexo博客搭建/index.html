hexo博客搭建

Posted on 2021-12-29 | In hexo博客搭建

安装git

Git是目前世界上最先进的分布式版本控制系统，可以有效、高速的处理从很小到非常大的项目版本管理。也就是用来管理你的hexo博客文章，上传到GitHub的工具。

到git官网上下载,Download git,下载后会有一个Git Bash的命令行工具，以后就用这个工具来使用git。

安装nodejs

Hexo是基于nodeJS编写的，所以需要安装一下nodeJs和里面的npm工具。

windows：nodejs选择LTS版本就行了。

安装完后执行：

node -v
npm -v

检查一下是否安装成功。

安装hexo

前面git和nodejs安装好后，就可以安装hexo了，你可以先创建一个文件夹blog，然后cd到这个文件夹下（或者在这个文件夹下直接右键git bash打开）。

输入命令

npm install -g hexo-cli

依旧用hexo -v查看一下版本

至此就全部安装完了。

接下来初始化一下hexo

hexo init myblog

这个myblog可以自己取什么名字都行，然后

cd myblog //进入这个myblog文件夹
npm install

新建完成后，指定文件夹目录下有：

  • node_modules: 依赖包
  • public：存放生成的页面
  • scaffolds：生成文章的一些模板
  • source：用来存放你的文章
  • themes：主题
  • ** _config.yml: 博客的配置文件**

打开hexo服务器：

hexo g
hexo server

在浏览器输入localhost:4000就可以看到生成的博客了。

使用ctrl+c可以把服务关掉。

GitHub建立个人仓库

首先注册一个GitHub账号

登录，有一个New repository，创建仓库

创建一个和你用户名相同的仓库，后面加.github.io，只有这样，将来要部署到GitHub page的时候，才会被识别，也就是xxxx.github.io，其中xxx就是你注册GitHub的用户名。

生成SSH添加到GitHub

在git bash中：

git config –global user.name “yourname”
git config –global user.email “youremail”

yourname是自己的GitHub用户名，youremail是邮箱

用以下的命令检查是否输对：

git config user.name
git config user.email

然后创建SSH，一路回车

ssh-keygen -t rsa -C “youremail”

这个时候它会告诉你已经生成了.ssh的文件夹。在你的电脑中找到这个文件夹。

ssh，简单来讲，就是一个秘钥，其中，id_rsa是你这台电脑的私人秘钥，不能给别人看的，id_rsa.pub是公共秘钥，可以随便给别人看。把这个公钥放在GitHub上，这样当你链接GitHub自己的账户时，它就会根据公钥匹配你的私钥，当能够相互匹配时，才能够顺利的通过git上传你的文件到GitHub上。

而后在GitHub的setting中，找到SSH keys的设置选项，点击New SSH key
把你的id_rsa.pub里面的信息复制进去。

在gitbash中，查看是否成功

命令：ssh -T git@github.com

将hexok部署到GitHub

如果你一切都配置好了，发布上传很容易，一句hexo d就搞定，当然关键还是你要把所有东西配置好。

首先，ssh key肯定要配置好。

其次，配置_config.yml中有关deploy的部分：

正确写法：

deploy:
type: git
repository: git@github.com:liuxianan/liuxianan.github.io.git
branch: master

错误写法：

deploy:
type: github
repository: https://github.com/liuxianan/liuxianan.github.io.git
branch: master

（当时被这个困扰了好久，一定要写成第一种….）

然后安装deploy-git ，也就是部署的命令,这样你才能用命令部署到GitHub。

命令：npm install hexo-deployer-git –save

然后

命令：

hexo clean
hexo generate
hexo deploy

其中 hexo clean清除了你之前生成的东西，也可以不加。 hexo generate 顾名思义，生成静态文章，可以用 hexo g缩写 hexo deploy 部署文章，可以用hexo d缩写 。

得到下图就说明部署成功了，过一会儿就可以在http://yourname.github.io 这个网站看到你的博客了

这样一个壳子就搭好了。

安装md编辑器

看网上许多大佬都用的typora，我也整个。

在csdn上找到一个大佬给的百度网盘，直接下载。

写博客

直接在git里输：hexo new post 博客名

就生成了相应的md文件，拖进typora编辑就行

关于写博客插入图片的问题

写博客自然要图文并茂，但是当我好不容易写完一篇的时候，高兴的上传，完事后，打开博客，图片竟然无法显示。

经过从网上找解决方法，终于搞定了。

首先安装图片插件

命令：npm install https://github.com/CodeFalling/hexo-asset-image

注：（网上好多教学是下载hexo-asset-image这个插件，但是我亲测不行，还得是上边那个）

找到post_asset_dolder,把这个选项从flase改为true

（一个一个找太慢，用ctrl+f搜索）

在typota文件选项，找到”偏好设置“选项

改成上边这样

然后创建博客，会同时生成一个与博客同名的文件夹，把你想上传到这篇博客里的图片先弄到这个文件夹里，然后直接拉进文章里

创建分类和标签

分类：

命令：hexo new page categories

在生成的文件夹添加：type： categories（注意格式，冒号后空格）

标签：

命令：hexo new page tags

同样的方法，在生成的tags文件夹添加type：tags

然后都保存

写文章时，给文章添加categories: 想写的分类
