链路聚合

X_T2_AGG1与X_T1_CORE1以及接入交换机之间部署链路聚合。

1.核心上创建eth-trunk2，将连接X_T2_AGG1的接口加入，并放通相应的vlan

2.在X_T2_AGG1上创建聚合组，将连接X_T1_CORE1的物理接口加入到聚合组中，然后放行相关的vlan。

3.在X_T2_AGG1上配置连接X_T2_ACC1的链路聚合

4.在x_T2_ AGG1上配置连接X_T2_ACC2的链路聚合:

5.判断X_ T2_ ACC1 和X_ T2_ ACC2上是如何连接到X_ T2_AGG1的，配置连接X_ T2_ AGG1的链路聚合，并放行所有vlan。

基础业务(VLAN、 IP、DHCP、路由)

在各个设备创建相应的vlan，配置vlanif，有线的网关在汇聚上，无线的网关在核心上，地址池已经配置好了，把地址池绑定到相应的vpn实例中即可，

vlan201用于X_T1_CORE 和X_T1_Export1 互联

vlan202用于X_T1_CORE 和X_T1_Export2 互联

vlan203用于X_T1_CORE 和X_T1_AC1互联

vlan204-207用于X_T1_CORE 和X_T1_FW1互联

vlan60用于X_T1_CORE 和服务器区互联

vlan208用于X_T1_CORE 和X_T1_AGG 互联

vlan209用于X_T1_CORE和X_T2_AGG互联

X_T1_CORE1 预配vlanif60，vlanif201，Vlanif203，v1anif208互联接口的地址，其他设备互联地址需要自行配置。

基础OSPF

两台出口路由器与和核心交换机之间的链路属于骨干网络，终端访问外网流量由出口两台路由器共同分担

根据网络隔离需求将内部网络与访客网络分别划分到OSPF的区域1与区域2

1.核心的public

2.核心的Employee实例（区域1）

3.核心的Guest实例（区域2）

4.X_T1_AGG1配置ospf对接核心，打通内网路由

5.X_T2_AGG1配置ospf对接核心

6.出口路由器的ospf

至此，核心可以与两个出口路由器建立邻居

核心与防火墙的OSPF

1.FW基础配置+虚墙配置+ip配置

2.根墙下配置防火墙的ospf

3.核心和防火墙互联接口二层配置

此时发现ospf不能完全建立，看下面⬇

①检查有没有undo vlan 1

②配置ospf的安全策略（两种方式选一种）,在虚墙视图下配置记得

③检查FW和核心连接的ospf这条命令有没有

vpn-instance-capability simple

④检查接口有没有绑定安全区域

⑤看接口有没有放行vlan

至此，FW和核心的ospf建立完成

4.过滤垃圾路由

现在在核心上查看路由表会发现：Employee中有Guest的路由，Guest有Employee的路由

原因：

核心的Employee实例中的路由通过206接口传给fw，fw通过Employee实例传给核心的Public（area 1），public的area 1会传给public的area 0，area 0会传给area 2，area 2连接fw的guest实例，就发给了fw的205，fw的guest再通过207传给核心的207，至此，核心的employee就受到了guest的路由。但是并不通，因为fw没有配置关于他们这些路由的安全策略，但是这垃圾路由占用路由表资源，所以要过滤掉。

5.配置nssa区域

出口

X_ T1_ Export2,采用双Internet线路实现上网分担

GE0/0/9采用静态IP (10.255.3.1/24) ，网关地址为10.255.3.254

GE0/0/10采用静态IP (10.255.4.1/24) ，网关地址为10.255.4.255

所有用户通过X_T1__Export2路由器访问Internet是需要通过NAT替换私网IP地址

对所有通过防火墙策略的IP报文进行SIP和SPORT转换 //napt 或easy ip

Export2 的GE0/0/9使用当前接口IP作为转换的地址 //easy ip

Export2 的GE0/0/10使用地址池(10.255.4.2-10.255.4.100) 替换私网IP地址和端口 //napt

X园区网内服务器(10.1.60.101) 向外提供WEB服务(端口号80)，公网用户通过10.255.4.1的8081号端口访问该服务//nat server

出口二：

准入认证

由于接入交换机不支持802.1X和策略联动功能，但是在这里又要用到802.1X功能。所以需要将认证点和执行点都部署在汇聚交换机上，接入交换机只需要透传EAP报文即可。其他所有的准入配置在两台汇聚交换机上配置即可。

1.为了保证端到端安全，需要对除了服务器以外的所有有线终端做认证

1)有线终端中， PC支持802.1认证，打印机及IP电话等哑终端不支持，已知当前接入交换机不支持802.1X和策略联动

2)汇聚交换机与认证服务器之间采用RADIUS协议进行交互，服务器地址为10.1.60.2，使用默认端口号进行认证，授权和计费，密钥为Huawei@123

3)所有设计到的模板等均命名为Employee。

4)认证域为 employee,且不允许用户使用其他域

5)RADIUS 服务器通过VLAN池名称方式下发用户的VLAN信息、VLAN池命令见表1-1

2.对于所有AP设备需要根据AP的MAC地址强制其到AP_ noauthen域进行认证，且认证模式为none.

3.用户信息与对于的授权信息已在认证与授权服务器上做配置,考试无需关心，用户名和密码信息: hr/Huawei@123、market/Huawei@123、procure/Huawei@123、 finance/Huawei@123

安全策略

1.防火墙上配置安全策略实现用户访问权限的控制

1)仅采购部、市场部、内部无线用户、外部无线用户可以访问internet

2)内部无线用户仅可以访问服务器网段的某一服务器, IP为10.1.60.100

3)外部无线用户仅可以访问服务器中的HTTP服务(服务名为Guest Service) IP 地址为10.1.60.99, 端口号为3389(TCP)

2.安全策略要求

1)必须包含源目安全区域

2)若包含特定服务需使用系统预定义服务或自定义服务(Service-set) 表示

3.外部无线用户访问Guest Service服务的流量直接在2个虚拟系统之间转发

1)Guest 和Employee的virtual-if属于untrust安全域，IP 地址分别为10.1.200.254/32与10.1 .200.253/32

2)仅当外部无线用户访问10.1.60.99时，流量直接在虚拟系统之间转发
